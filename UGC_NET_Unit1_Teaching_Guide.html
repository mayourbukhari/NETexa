Mathematical Logic

Propositional Logic · Predicate Logic · Normal Forms · Rules of Inference

🌱 What is Mathematical Logic?

Logic is the science of correct reasoning. In computers, it helps us build conditions, circuits, and proofs. Think of it as the grammar of mathematics.

Real-life analogy

Just like "If it rains, I carry an umbrella" — logic deals with if-then relationships and whether statements are TRUE or FALSE.

📌 Propositional Logic — The Basics

A proposition is any statement that is either TRUE (T) or FALSE (F). Not both, not neither.

✅ Valid Propositions
  • "2 + 2 = 4" → TRUE
  • "Delhi is in UK" → FALSE
  • "All birds fly" → FALSE
❌ NOT Propositions
  • "What time is it?" (question)
  • "Close the door!" (command)
  • "x + 1 = 5" (variable — unknown)

Two formulas are logically equivalent (≡) if they have the same truth value for all inputs.

De Morgan's Laws: ¬(p ∧ q) ≡ ¬p ∨ ¬q ¬(p ∨ q) ≡ ¬p ∧ ¬q

📌 Normal Forms — CNF & DNF

CNF — Conjunctive Normal Form

AND of ORs (clauses)
Example: (p ∨ q) ∧ (¬p ∨ r)

Think: AND outside, OR inside

DNF — Disjunctive Normal Form

OR of ANDs (minterms)
Example: (p ∧ q) ∨ (¬p ∧ r)

Think: OR outside, AND inside

💡 Memory Trick

CNF = Clubs (∧) of Friends (∨)  |  DNF = Divisions (∨) of Names (∧)

📌 Predicate Logic (First Order Logic)

Propositional logic can't say "All students passed". For this, we need predicates and quantifiers.

∀ Universal Quantifier

∀x P(x) = "For ALL x, P(x) is true"

Example: ∀x (Human(x) → Mortal(x))
"All humans are mortal"

∃ Existential Quantifier

∃x P(x) = "There EXISTS some x"

Example: ∃x (Student(x) ∧ Passed(x))
"Some student passed"

Which of the following is NOT a proposition?
✅ Answer: B — "Please sit down" is a command, not a declarative statement. It cannot be assigned a truth value. Commands, questions, and exclamations are NOT propositions.

The truth value of (T → F) is:
✅ Answer: C — For implication (p → q), the ONLY case it is FALSE is when p = TRUE and q = FALSE. Here p=T, q=F, so T→F = FALSE. Remember: A liar is one who promises (T) but doesn't deliver (F).
Question 3 of 10
Which law states: ¬(p ∧ q) ≡ ¬p ∨ ¬q ?
✅ Answer: A — De Morgan's First Law: ¬(p ∧ q) ≡ ¬p ∨ ¬q. De Morgan's Second Law: ¬(p ∨ q) ≡ ¬p ∧ ¬q. These are extremely important for NET exam!
Question 4 of 10
A formula that is TRUE for all possible truth assignments is called:
✅ Answer: D — A Tautology is always TRUE (e.g., p ∨ ¬p). A Contradiction is always FALSE (e.g., p ∧ ¬p). A Contingency is sometimes true, sometimes false.
Question 5 of 10
The rule of inference "p → q, ¬q ∴ ¬p" is called:
✅ Answer: B — Modus Tollens: "If p then q; q is false; therefore p is false." Example: If it rains, ground is wet. Ground is NOT wet. Therefore it did NOT rain.
Question 6 of 10
∀x ∃y (x + y = 0) over integers means:
✅ Answer: C — ∀x means "for every x". ∃y means "there exists some y (which may depend on x)". So for every integer x, we can find y = -x such that x + y = 0. This is TRUE over integers.
Question 7 of 10
Which of the following is equivalent to p → q ?
✅ Answer: A — p → q ≡ ¬p ∨ q. This is a fundamental equivalence. Verify: p→q is false only when p=T,q=F. ¬p∨q is also false only when p=T,q=F. They match!
Question 8 of 10
In CNF (Conjunctive Normal Form), the formula is a:
✅ Answer: D — CNF = AND of OR clauses. Example: (p∨q) ∧ (¬p∨r). DNF is the opposite: OR of AND clauses. Example: (p∧q) ∨ (¬p∧r).
Question 9 of 10
p ↔ q is logically equivalent to:
✅ Answer: D — p↔q means "p if and only if q". It equals (p→q)∧(q→p) AND also equals (p∧q)∨(¬p∧¬q). Both B and C are correct equivalences!
Question 10 of 10
The negation of "Some students are intelligent" is:
✅ Answer: C — "Some students are intelligent" = ∃x (Student(x) ∧ Intelligent(x)). Its negation = ∀x ¬(Student(x) ∧ Intelligent(x)) = ∀x (¬Student(x) ∨ ¬Intelligent(x)) = "No student is intelligent".

Sets & Relations

Set Operations · Equivalence Relations · Partial Ordering

📌 Sets — The Foundation

A set is a well-defined collection of distinct objects called elements.

Key Set Operations
  • Union A∪B — All elements in A OR B (or both)
  • Intersection A∩B — Only elements in BOTH A and B
  • Difference A−B — Elements in A but NOT in B
  • Complement Ā — All elements NOT in A (in universal set U)
  • Power Set P(A) — Set of ALL subsets of A; |P(A)| = 2^|A|
✏️ Solved Example

A = {1,2,3,4}, B = {3,4,5,6}. Find A∪B, A∩B, A−B.

A∪B = {1,2,3,4,5,6} ← All elements combined A∩B = {3,4} ← Only common elements A−B = {1,2} ← In A but not in B |P(A)| = 2^4 = 16 ← Total subsets of A

📌 Relations

A relation R from set A to set B is a subset of A×B (Cartesian product).

A relation on A is a subset of A×A. For a relation R on set A:

PropertyMeaningExample
Reflexive(a,a) ∈ R for all a ∈ Aa = a (equality)
SymmetricIf (a,b)∈R then (b,a)∈Ra is sibling of b ↔ b is sibling of a
Antisymmetric(a,b)∈R and (b,a)∈R → a=ba ≤ b and b ≤ a → a = b
Transitive(a,b)∈R, (b,c)∈R → (a,c)∈Ra<b, b<c → a<c
💡 Key Definitions

Equivalence Relation = Reflexive + Symmetric + Transitive (e.g., "same age")
Partial Order = Reflexive + Antisymmetric + Transitive (e.g., ≤ on integers)

Counting, Induction & Probability

Pigeonhole · Permutations · Combinations · Bayes' Theorem

📌 Pigeonhole Principle

If n+1 objects are placed in n boxes, at least one box contains ≥ 2 objects.

✏️ Solved Example

Q: In a class of 367 students, at least how many share the same birthday?

Solution: There are 366 possible birthdays (including Feb 29). We have 367 students (pigeons) and 366 days (holes). By Pigeonhole Principle, at least ⌈367/366⌉ = 2 students share a birthday.

📌 Permutations & Combinations

Permutation (order matters): P(n,r) = n! / (n-r)! Combination (order doesn't): C(n,r) = n! / [r! × (n-r)!]
✏️ Solved Example

Q: In how many ways can a team of 3 be chosen from 7 people? And in how many ways can a President, VP, Secretary be appointed from 7?

Team (Combination): C(7,3) = 7!/(3!×4!) = 35 ways Officers (Permutation): P(7,3) = 7!/4! = 7×6×5 = 210 ways
💡 Memory Trick

Combination = Choose (like a committee — nobody has a special title)
Permutation = Position matters (like a race — 1st, 2nd, 3rd)

📌 Bayes' Theorem

P(A|B) = P(B|A) × P(A) / P(B)
✏️ Solved Example

A disease affects 1% of population. Test is 95% accurate (sensitivity), 90% specific. You test positive. What's the probability you actually have the disease?

P(Disease) = 0.01, P(No Disease) = 0.99 P(+|Disease) = 0.95, P(+|No Disease) = 0.10 P(Disease|+) = (0.95 × 0.01) / [(0.95×0.01) + (0.10×0.99)] = 0.0095 / (0.0095 + 0.099) = 0.0095 / 0.1085 ≈ 8.76%

⚡ Surprising! Even with a positive test, only ~9% chance of having the disease (because it's rare).

Group Theory

Groups · Subgroups · Rings · Fields · Isomorphism

📌 What is a Group?

A Group (G, *) is a set G with a binary operation * satisfying 4 properties:

#PropertyMeaningExample (Z, +)
1Closurea*b ∈ G for all a,b ∈ G2+3=5 ∈ Z ✓
2Associativity(a*b)*c = a*(b*c)(1+2)+3=1+(2+3) ✓
3Identity∃e: a*e = e*a = aa+0=a, e=0 ✓
4Inverse∀a ∃a⁻¹: a*a⁻¹ = e3+(-3)=0 ✓
Important Types
  • Semigroup: Closure + Associativity (no identity/inverse needed)
  • Monoid: Semigroup + Identity
  • Group: Monoid + Inverse
  • Abelian (Commutative) Group: Group + a*b = b*a
💡 Memory Trick — CAII

Closure → Associativity → Identity → Inverse = Group!
Add Commutativity → Abelian Group

📌 Rings and Fields

Ring (R, +, ×)
  • (R, +) is Abelian Group
  • (R, ×) is Semigroup (closed + associative)
  • Distributive: a×(b+c) = a×b + a×c
  • Example: (Z, +, ×) integers
Field (F, +, ×)
  • Ring + multiplicative inverse for non-zero elements
  • Both + and × form Abelian Groups
  • Examples: (Q, +, ×), (R, +, ×), (Z_p for prime p)

Graph Theory

Paths · Eulerian · Hamiltonian · Trees · Spanning Trees

📌 Graph Basics

A Graph G = (V, E) consists of Vertices (nodes) and Edges (connections).

Types of Graphs
  • Simple Graph: No self-loops, no multiple edges
  • Multigraph: Multiple edges between same nodes
  • Weighted Graph: Edges have weights/costs
  • Directed Graph: Edges have direction
  • Bipartite: Vertices split in 2 groups; edges only between groups
Key Formulas
  • Sum of degrees = 2 × |E|
  • In simple graph: max edges = n(n-1)/2
  • Handshaking Lemma: #odd-degree vertices is even
  • Tree with n vertices has n-1 edges
  • Planar: V − E + F = 2 (Euler's formula)

📌 Trees

A tree is a connected, acyclic (no cycles) graph.

Properties of a Tree with n vertices
  • Has exactly n−1 edges
  • There is exactly one path between any two vertices
  • Adding any edge creates exactly one cycle
  • Removing any edge disconnects the graph
✏️ Tree Traversals (for Rooted Trees)
Tree: A / \ B C / \ D E Pre-order (Root→Left→Right): A B D E C In-order (Left→Root→Right): D B E A C Post-order (Left→Right→Root): D E B C A

📌 Karnaugh Map (K-Map) — Simplification

K-maps are visual tools to minimize Boolean functions. Group 1s in powers of 2 (1, 2, 4, 8...) to simplify.

✏️ Solved Example — 2-Variable K-Map

Simplify: F(A,B) = Σ(1,2,3) — minterms 1, 2, 3

K-Map: B=0 B=1 A=0 | 0(F) | 1(T) | A=1 | 2(T) | 3(T) | Group {1,3}: B column (A changes) → B Group {2,3}: A row (B changes) → A F = A + B

✅ Simplified from 3 minterms to just F = A + B

💡 K-Map Rules

• Groups must be rectangles of size 1, 2, 4, 8... (powers of 2)
• Groups CAN wrap around edges
• Each 1 must be in at least one group
• Larger groups = simpler expression
• Find the minimum number of largest groups

Optimization

Linear Programming · Simplex Method · PERT-CPM

📌 Linear Programming (LP)

LP finds the maximum or minimum of a linear objective function subject to linear constraints.

Standard LP Form

Maximize (or Minimize): Z = c₁x₁ + c₂x₂ + ... (Objective Function)
Subject to: Linear inequalities (Constraints)
With: x₁, x₂ ≥ 0 (Non-negativity)

✏️ Graphical Method — Solved Example

Maximize Z = 3x + 5y, subject to: x ≤ 4, y ≤ 6, 3x + 2y ≤ 18, x,y ≥ 0

1
Find corner points

Plot constraints. Find vertices of feasible region: (0,0), (4,0), (4,3), (2,6), (0,6)

2
Evaluate Z at each corner

Z(0,0)=0 | Z(4,0)=12 | Z(4,3)=27 | Z(2,6)=36 | Z(0,6)=30

3
Optimal solution

Maximum Z = 36 at point (2,6) ✅

📌 PERT and CPM

CPM — Critical Path Method
  • Deterministic activity times
  • Used for construction, manufacturing
  • Critical Path = Longest path = Minimum project duration
  • Activities on critical path have ZERO float/slack
PERT — Program Evaluation Review Technique
  • Probabilistic (3 time estimates)
  • t_e = (O + 4M + P) / 6
  • O=optimistic, M=most likely, P=pessimistic
  • Used for R&D, software projects
✏️ Critical Path Example
Activities: A(3), B(4), C(2), D(5), E(3) Network: Start→A→C→E→End (path: 3+2+3=8) Start→B→D→End (path: 4+5=9) ← CRITICAL PATH Critical Path = B→D = 9 days (longest) Float of A = 9 - 8 = 1 day (can delay by 1 day) Float of B, D = 0 (critical — no delay allowed!)
