Programming Paradigms

Procedural · Object-Oriented · Functional · Declarative

🌍 What are Programming Paradigms?

A programming paradigm is a fundamental style or approach to writing code. It represents how we think about organizing and solving problems through code.

Analogy

Think of paradigms as different architectural styles (Gothic, Modern, Classical). Each has different principles, but they all serve the same purpose — building structures.

1️⃣ Procedural Programming

Procedural Programming Basics

Code flows as a sequence of procedures or functions. Data and logic are separate. You tell the computer HOW to do things step-by-step.

  • Languages: C, Pascal, FORTRAN, BASIC
  • Core Idea: Break problem into functions
  • Data Organization: Global/local variables passed to functions
Key Characteristics
  • Sequential execution with loops and conditionals
  • Functions/subroutines for code reuse
  • Focus on algorithms and control flow
  • Shared global data (less encapsulation)

Procedural Example: Calculate Factorial

// C - Procedural Style int factorial(int n) { if (n <= 1) return 1; return n * factorial(n-1); } int main() { int result = factorial(5); // 120 printf("%d", result); return 0; }
Analysis

Procedural thinking: "To get factorial, call this function in sequence → recursion → return result"

2️⃣ Object-Oriented Programming (OOP)

OOP Core Concepts

Code is organized around objects — data and functions bundled together. You model real-world entities as classes and interact with objects.

  • Languages: Java, C++, Python, C#, Ruby
  • Core Idea: Everything is an object with properties (state) and methods (behavior)

🔑 Four Pillars of OOP

1. Encapsulation

Bundling data & methods together. Hide internal details with access modifiers (public/private/protected).

class BankAccount { private double balance; public void deposit(double amt) { balance += amt; } }

2. Inheritance

Child class inherits properties/methods from parent class. Enables code reuse and hierarchical relationships.

class Animal { ... } class Dog extends Animal { void bark() { ... } }

🔑 (continued...)

3. Polymorphism

Same interface, multiple implementations. Method overriding allows behavior to differ in subclasses.

class Dog extends Animal { void makeSound() { System.out.println("Woof!"); } } class Cat extends Animal { void makeSound() { System.out.println("Meow!"); } }

4. Abstraction

Hiding complexity. Show only essential features. Use abstract classes & interfaces.

abstract class Shape { abstract void draw(); } class Circle extends Shape { void draw() { ... } }

3️⃣ Functional Programming

Functional Programming Principles

Code treats computation as evaluation of mathematical functions. Emphasizes immutability and avoids changing state.

  • Languages: Lisp, Haskell, Scheme, Elixir, Clojure
  • Modern Support: Python, JavaScript, Java (lambdas), Kotlin
Core Principles
  • Pure Functions: Same input → same output, no side effects
  • Immutability: Data doesn't change after creation
  • First-Class Functions: Functions as values (passed as arguments, returned)
  • Higher-Order Functions: Functions that take/return other functions
  • Recursion: Primary looping mechanism instead of mutations

Functional Example: Map, Filter, Reduce

// JavaScript - Functional Style const numbers = [1, 2, 3, 4, 5]; // Map: Transform each element const doubled = numbers.map(n => n * 2); // [2, 4, 6, 8, 10] // Filter: Keep elements that satisfy a condition const evens = numbers.filter(n => n % 2 === 0); // [2, 4] // Reduce: Combine elements into single value const sum = numbers.reduce((acc, n) => acc + n, 0); // 15
💡 Key Difference: Procedural vs OOP vs Functional

Procedural: "How to calculate sum? Loop through array, add each element"
OOP: "Create a List object with sum() method that knows how to add"
Functional: "Apply reduce function to sum all elements without changing state"

Data Structures

Arrays · Linked Lists · Stacks · Queues · Trees · Graphs · Hash Tables
What is a Data Structure?

A data structure is a specialized format for organizing, processing, and storing data. The right data structure can dramatically improve algorithm efficiency.

Linear Data Structures

Arrays - Static Linear Collections

A fixed-size collection storing elements of the same type in contiguous memory.

Characteristics
  • Access: O(1) — Direct access via index
  • Search: O(n) — Linear search; O(log n) if sorted (binary search)
  • Insert/Delete: O(n) — Requires shifting elements
  • Space: O(n)
int arr[5] = {10, 20, 30, 40, 50}; // Access in O(1) int x = arr[2]; // 30 // Insert 25 at index 2 — O(n) operation // {10, 20, 25, 30, 40, 50}

Linked Lists - Dynamic Linear Structures

Nodes connected via pointers/references. Unlike arrays, can grow/shrink dynamically.

Singly Linked List:

[10]→[20]→[30]→[40]→NULL

Access: O(n)
Insert at head: O(1)
Insert at position: O(n)
Delete: O(n)

Doubly Linked List:

NULL←[10]⇄[20]⇄[30]→NULL

Reverse traversal: O(n)
More flexibility but more memory per node

struct Node { int data; Node* next; }; // Insert at beginning: O(1)

Stacks - LIFO (Last In, First Out)

Imagine a stack of plates — you add/remove from the top. Perfect for recursion, undo operations, expression evaluation.

[3] ← TOP (pop/peek) [2] [1] BOTTOM
Operations
  • Push(x): Add x to top — O(1)
  • Pop(): Remove from top — O(1)
  • Peek(): View top — O(1)
  • isEmpty(): O(1)
class Stack { push(x) { this.arr.push(x); } pop() { return this.arr.pop(); } peek() { return this.arr[this.arr.length-1]; } } // Expression evaluation: "3 + 4 * 2" // Check balanced parentheses // DFS traversal

Queues - FIFO (First In, First Out)

Like a line at a store — first person entered is first to leave. Used in scheduling, BFS, printer queues.

ENQUEUE → [1][2][3] ← DEQUEUE HEAD/FRONT REAR/BACK
Operations
  • Enqueue(x): Add to rear — O(1)
  • Dequeue(): Remove from front — O(1)
  • Front(): View first elem — O(1)
  • isEmpty(): O(1)

Non-Linear Data Structures

Trees - Hierarchical Structures

Nodes arranged in parent-child hierarchy. Root has no parent; leaves have no children.

ROOT (A) / \ B C / \ / D E F
  • Binary Tree: Each node has ≤ 2 children
  • Binary Search Tree (BST): Left child < Parent < Right child
  • Balanced Trees: AVL, Red-Black Trees keep height O(log n)
Tree Traversals

In-Order (Left-Root-Right): For BST gives sorted sequence
Pre-Order (Root-Left-Right): Root first, useful for copying tree
Post-Order (Left-Right-Root): Root last, useful for deletion
Level-Order: BFS — process by levels

Hash Tables - Fast Lookups

Maps keys to values using hash function. Provides average O(1) access.

How Hash Tables Work

Hash function transforms key → index in array. Example: h(key) = key % table_size

// Collision handling: Chaining int table[10]; // Multiple elements at same hash address stored as linked list // Common use: Dictionary, Cache, Symbol Table
Collision Handling Methods
  • Chaining: Store multiple items at same index using linked list
  • Open Addressing: Find another empty slot (linear probe, quadratic probe, double hashing)

Graphs - General Network Structure

Vertices (nodes) connected by edges. Can be directed/undirected, weighted/unweighted.

Directed Graph: A→B (edges have direction)

Undirected Graph: A—B (two-way connection)

Weighted Graph: Edges have costs/distances

Unweighted Graph: All edges equal

Representations
  • Adjacency Matrix: Space O(V²), access O(1) — good for dense graphs
  • Adjacency List: Space O(V+E), traverse O(V+E) — good for sparse graphs

Algorithm Analysis

Time Complexity · Space Complexity · Big-O Notation · Asymptotic Analysis

⏱️ What is Algorithm Analysis?

Algorithm analysis measures how efficiently an algorithm uses computational resources — primarily time and space — as input size grows.

Why Does It Matter?
  • Two algorithms solving same problem, different speeds
  • For n=10 vs n=1,000,000 → difference is crucial
  • Helps choose between data structures

📊 Big-O Notation (Asymptotic Analysis)

Describes how algorithm performance grows with input size. Focuses on worst-case upper bound.

NotationNameExampleImpression
O(1)ConstantArray access [i]Perfect ✓
O(log n)LogarithmicBinary SearchExcellent ✓
O(n)LinearLinear SearchGood
O(n log n)LinearithmicMerge Sort, Quick SortAcceptable
O(n²)QuadraticBubble Sort, Nested loopsFair ⚠️
O(n³)Cubic3 nested loopsPoor ✗
O(2ⁿ)ExponentialSubset generationVery Poor ✗
O(n!)FactorialPermutationsExtremely Poor ✗
Visual Comparison

n=10: O(n²)=100, O(n³)=1000, O(2ⁿ)=1024
n=20: O(n²)=400, O(n³)=8000, O(2ⁿ)=1M
n=100: O(n²)=10K, O(n³)=1M, O(2ⁿ)=10³⁰ (impossible!)

📈 How to Calculate Time Complexity

1
Count Basic Operations

Find loops, recursive calls, nested structures. Each costs O(1).

2
For Simple Loops: Multiply Count

Loop running n times → O(n). Nested loop n×n → O(n²).

3
Drop Constants & Lower Terms

O(2n + 5) = O(n). O(n² + n) = O(n²). Keep only highest order.

Examples

// Example 1: Simple loop — O(n) for (int i = 0; i < n; i++) { print(arr[i]); // O(1) operation, runs n times } // Total: O(n) // Example 2: Nested loops — O(n²) for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { print(arr[i][j]); // O(1), runs n×n times } } // Total: O(n²) // Example 3: Halving input — O(log n) while (n > 1) { n = n / 2; // Loop runs log₂(n) times } // Total: O(log n)

💾 Space Complexity

Measures extra memory used by algorithm (excluding input). Calculated similarly to time.

Common Space Complexities
  • O(1): Constant extra space (swaps, counters)
  • O(n): Linear — creating array of size n
  • O(log n): Recursion depth (binary search call stack)
  • O(n²): 2D array of size n×n
// Space O(n) — temp array int[] temp = new int[n]; for (int i = 0; i < n; i++) { temp[i] = arr[i]; } // Space O(1) — in-place swap int x = arr[0]; arr[0] = arr[n-1]; arr[n-1] = x;
⚠️ Important: Trade-offs

Time vs Space: Often inverse. Caching increases space to reduce time. Recursion may reduce time but increase space (call stack).

Sorting Algorithms

Bubble · Selection · Insertion · Merge · Quick · Heap · Counting · Radix
Why Study Sorting?

Sorting is fundamental — enables faster searching, provides organized data, often subproblem of larger algorithms. Understanding sorting teaches complexity analysis.

Simple Sorts (Inefficient but Educational)

Bubble Sort - The Simple Approach

Repeatedly swap adjacent elements if they're in wrong order. "Bubbles" largest element to end each pass.

Time (Best)Time (Avg)Time (Worst)SpaceStable?
O(n)O(n²)O(n²)O(1)✓ Yes
void bubbleSort(int[] arr) { for (int i = 0; i < arr.length; i++) { for (int j = 0; j < arr.length-i-1; j++) { if (arr[j] > arr[j+1]) { // Swap int temp = arr[j]; arr[j] = arr[j+1]; arr[j+1] = temp; } } } } // Input: [5, 2, 8, 1, 9] // Pass 1: [2, 5, 1, 8, 9] // Pass 2: [2, 1, 5, 8, 9] // Pass 3: [1, 2, 5, 8, 9] ✓

Selection Sort - Divide & Minimize

Partition array into sorted/unsorted. Repeatedly find minimum in unsorted, move to sorted.

Time (All Cases)SpaceStable?
O(n²)O(1)✗ No
void selectionSort(int[] arr) { for (int i = 0; i < arr.length; i++) { int minIdx = i; for (int j = i+1; j < arr.length; j++) if (arr[j] < arr[minIdx]) minIdx = j; // Swap minimum with position i int temp = arr[i]; arr[i] = arr[minIdx]; arr[minIdx] = temp; } }

Insertion Sort - Efficient for Small Arrays

Build sorted array element by element. Insert next element into correct position in sorted portion.

Time (Best)Time (Avg)Time (Worst)Space
O(n)O(n²)O(n²)O(1)
void insertionSort(int[] arr) { for (int i = 1; i < arr.length; i++) { int key = arr[i]; int j = i-1; while (j >= 0 && arr[j] > key) { arr[j+1] = arr[j]; j--; } arr[j+1] = key; } }
Best Use Case

Nearly sorted arrays, small arrays (< 50 elements), online sorting

Efficient Sorts (Production Quality)

Merge Sort - Divide & Conquer Giant

Divide array in half recursively, then merge sorted halves. Guaranteed O(n log n).

Time (All Cases)SpaceStable?
O(n log n)O(n)✓ Yes
Algorithm Steps
1
Divide array at midpoint
2
Recursively sort left and right halves
3
Merge sorted halves maintaining order
void mergeSort(int[] arr, int left, int right) { if (left < right) { int mid = left + (right - left) / 2; mergeSort(arr, left, mid); mergeSort(arr, mid+1, right); merge(arr, left, mid, right); // Merge sorted parts } } // Example: [5,2,8,1] → [2,1,5,8] → [1,2,5,8]

Quick Sort - Average Case King

Partition array using pivot, recursively sort partitions. Average O(n log n), worst O(n²).

Time (Best)Time (Avg)Time (Worst)Space
O(n log n)O(n log n)O(n²)O(log n)*

* Call stack depth

Smart Pivot Selection
  • Random pivot → expected O(n log n)
  • Median-of-three → reduces worst case probability
  • Bad pivot → O(n²) (avoid!)

Heap Sort - Space Efficient & Guaranteed

Build max heap, repeatedly extract max. O(n log n) guaranteed, O(1) space.

Time (All Cases)SpaceStable?
O(n log n)O(1)✗ No
Advantage

Worst-case guaranteed unlike Quick Sort. Used in priority queues, heap-based algorithms.

Specialized Sorts (Non-Comparative)

Counting Sort - For Integer-Keyed Data

Count occurrences of each value, reconstruct sorted array. O(n + k) where k = range of values.

When to Use
  • Small range of integers: k ≈ n
  • Not comparative — doesn't use comparisons
  • Example: Sort 1 million numbers in range 1-100

Radix Sort - Multi-Digit Integers

Sort by each digit/position from least to most significant. O(d × (n + k)) where d = digit positions.

Use Case

Sorting integers with many digits, string sorting, practical for huge datasets with small k.

🎯 Sorting Algorithm Comparison Guide

Quick Sort: Industry standard (Java, C++). Fast in practice despite O(n²) worst case.
Merge Sort: When guaranteed O(n log n) needed & stability matters. Preferred for linked lists.
Heap Sort: When worst-case must be O(n log n) & space critical.
Counting/Radix: Restricted domains, huge datasets.

Searching & Tree Structures

Linear Search · Binary Search · BST · AVL · B-Trees · Hashing

🔎 Linear Search - Brute Force

Check each element sequentially until found. Simple but O(n).

TimeSpaceData Requirement
O(n)O(1)Unsorted works fine
int linearSearch(int[] arr, int target) { for (int i = 0; i < arr.length; i++) if (arr[i] == target) return i; return -1; // Not found }

⚡ Binary Search - Efficient Log Approach

Repeatedly divide sorted array in half. O(log n) — exponentially faster than linear!

TimeSpaceRequirement
O(log n)O(1)MUST be sorted
int binarySearch(int[] arr, int target) { int left = 0, right = arr.length - 1; while (left <= right) { int mid = left + (right - left) / 2; if (arr[mid] == target) return mid; else if (arr[mid] < target) left = mid + 1; else right = mid - 1; } return -1; }
💡 Why Mid = left + (right-left)/2?

Prevents integer overflow! (left + right) might exceed MAX_INT, but (right - left)/2 is safe.

🌳 Binary Search Trees (BST)

Each node: Left subtree < node < Right subtree. Enables O(log n) search in balanced tree.

BST Operations
  • Search: O(log n) average, O(n) worst (unbalanced)
  • Insert: Find position, add node — O(log n) average
  • Delete: Requires rebalancing — O(log n) average
8 / \ 5 12 / \ / \ 3 7 10 15

In-order traversal (3,5,7,8,10,12,15): Gives sorted sequence from BST!

⚖️ Balanced Trees - AVL & Red-Black

Self-balancing BSTs maintain height O(log n), guaranteeing O(log n) operations always.

AVL Tree

Height balance: Height difference ≤ 1

Rotations: Rebalance via rotations on insert/delete

Red-Black Tree

Color property: Nodes colored red/black

Fewer rotations: Less overhead than AVL

🌲 B-Trees - Databases Love These

Multi-way trees where each node can have many children. Used in databases, file systems.

B-Tree Properties
  • Each node has ≤ m children (m = order)
  • All leaves at same level
  • Minimizes disk accesses (groups keys together)
  • Perfect for external storage

Design Patterns

Creational · Structural · Behavioral patterns for scalable software
What are Design Patterns?

Reusable solutions to common software design problems. Proven blueprints for creating flexible, maintainable code. Like architectural patterns for buildings.

Creational Patterns (Object Creation)

Singleton Pattern

Single instance of a class, globally accessible. Prevents multiple instantiation.

Use Cases

Database connection pool, Logger, Configuration manager, Thread pool

class Singleton { private static Singleton instance; private Singleton() {} // Private constructor public static Singleton getInstance() { if (instance == null) instance = new Singleton(); return instance; } } // Usage: Singleton s = Singleton.getInstance();

Factory Pattern

Create objects without specifying exact classes. Encapsulates object creation logic.

Benefit

Decouples creation from usage. Easy to add new types without changing client code.

class DatabaseFactory { public static Database createDB(String type) { if (type.equals("mysql")) return new MySQLDatabase(); else if (type.equals("mongodb")) return new MongoDatabase(); return null; } } // Usage: Database db = DatabaseFactory.createDB("mysql");

Behavioral Patterns (Object Interaction)

Observer Pattern

Subjects notify multiple observers of state changes. Enables loose coupling.

Real-world Example

Event listeners in UI — button click notifies multiple handlers

interface Observer { void update(String event); } class Button { private List<Observer> observers; void click() { for (Observer obs : observers) obs.update("Button Clicked"); // Notify all } }

Strategy Pattern

Different algorithms interchangeable at runtime. Choose strategy dynamically.

Use Case

Payment methods (Credit Card, PayPal, Bitcoin), Sorting strategies

interface PaymentStrategy { void pay(double amount); } class CreditCardPayment implements PaymentStrategy { public void pay(double amount) { System.out.println("Pay " + amount + " via Credit Card"); } } // Runtime selection: PaymentStrategy strategy = new CreditCardPayment(); strategy.pay(100);

Advanced Topics

Dynamic Programming · Greedy · Divide & Conquer · Graph Algorithms

🔄 Dynamic Programming - Overlapping Subproblems

Solve complex problems by breaking into overlapping subproblems, reusing solutions (memoization).

When to Use DP
  • Problem has optimal substructure (optimal solution contains optimal subsolutions)
  • Problem has overlapping subproblems (recalculates same subproblems)
// Fibonacci with DP (Memoization) int memo[100]; int fib(int n) { if (n <= 1) return n; if (memo[n] != 0) return memo[n]; memo[n] = fib(n-1) + fib(n-2); return memo[n]; } // Without DP: O(2ⁿ) exponential // With DP: O(n) linear! (compute each once)

🎯 Greedy Algorithms - Local Optimization

Make locally optimal choice at each step. Works when greedy choice leads to global optimum.

Classic Examples
  • Huffman Coding: Build optimal prefix codes for compression
  • Activity Selection: Select maximum non-overlapping activities
  • Minimum Spanning Tree: Kruskal's & Prim's algorithms

✂️ Divide & Conquer - Split, Solve, Combine

Break problem into smaller independent subproblems, solve recursively, combine solutions.

Characteristics
  • Divide problem into independent parts
  • Recursively solve subproblems
  • Combine subsolutions
  • Usually O(n log n) complexity
Examples

Merge Sort, Quick Sort, Binary Search, Strassen's Matrix Multiplication

🕸️ Graph Algorithms

Depth-First Search (DFS)

void dfs(node) { visited[node] = true; for each neighbor: if (!visited[neighbor]) dfs(neighbor); } // Uses stack/recursion // Time: O(V+E)

Breadth-First Search (BFS)

void bfs(start) { Queue q; q.enqueue(start); while (!q.empty()) { node = q.dequeue(); for each neighbor: q.enqueue(neighbor); } } // Uses queue // Time: O(V+E)
Shortest Path Algorithms
  • Dijkstra: Single-source shortest paths, non-negative weights — O((V+E)log V)
  • Bellman-Ford: Handles negative weights — O(VE)
  • Floyd-Warshall: All pairs shortest paths — O(V³)

📝 Practice Questions

Question 1 of 15 0/15 (0%)
Question 1
Which of the following is NOT a programming paradigm?
Explanation: Paradigms are fundamental approaches (OOP, Functional, Procedural, Declarative). "Graphical programming" is a tool/interface, not a fundamental paradigm.
Question 2
What is the time complexity of binary search?
Explanation: Binary search repeatedly halves the search space. Each operation: n → n/2 → n/4 → ... → 1, requiring log₂(n) iterations.
Question 3
Which data structure uses LIFO (Last In, First Out) principle?
Explanation: Stacks use LIFO — last pushed element is first popped. Queues use FIFO (First In, First Out).
Question 4
What is the worst-case time complexity of quicksort?
Explanation: Worst case occurs with poor pivot selection (e.g., always picking smallest/largest). Average case is O(n log n), but worst is O(n²).
Question 5
In an AVL tree, what is maintained to ensure balance?
Explanation: AVL trees maintain a balance factor (height difference) of at most ±1 between left and right subtrees, ensuring O(log n) operations.
Question 6
Which sorting algorithm has the best average case time complexity?
Explanation: Both Merge Sort and Quick Sort have O(n log n) average case. Merge Sort is guaranteed; Quick Sort depends on pivot. Bubble/Selection are O(n²).
Question 7
What does Big-O notation describe?
Explanation: Big-O describes worst-case upper bound on growth rate as input size increases. For example, O(n²) means at most quadratic operations.
Question 8
Which pattern ensures only one instance of a class exists?
Explanation: Singleton pattern uses private constructor and static method to ensure only one instance. Used for loggers, database connections.
Question 9
What is the space complexity of merge sort?
Explanation: Merge sort requires temporary arrays during merge operations, necessitating O(n) extra space. Unlike quick sort which is O(log n).
Question 10
In a hash table, what is the average time complexity of search, insert, and delete?
Explanation: With good hash function and collision handling, average is O(1). Worst case (all collisions) is O(n), but rare with proper implementation.
Question 11
Which approach uses "optimal substructure" and "overlapping subproblems"?
Explanation: DP solves overlapping subproblems via memoization. Greedy doesn't need overlapping. D&C typically solves independent subproblems.
Question 12
What time complexity does BFS have for traversing a graph?
Explanation: BFS/DFS visit each vertex once (V) and traverse each edge once (E), resulting in O(V+E). Same as both traversal methods.
Question 13
Which of the following is a characteristic of pure functions in functional programming?
Explanation: Pure functions are deterministic (same input → same output) and have no side effects. They're the foundation of functional programming.
Question 14
What is the main advantage of encapsulation in OOP?
Explanation: Encapsulation bundles data and methods, using access modifiers to control exposure. Improves maintainability and prevents misuse.
Question 15
Which algorithm guarantees O(n log n) in worst case without using extra space?
Explanation: Heap Sort guarantees O(n log n) (not O(n²) like Quick Sort worst case) and uses O(1) extra space (in-place). Merge Sort needs O(n) extra space.
